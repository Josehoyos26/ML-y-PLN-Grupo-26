Elaborado por: Hctor David Castro, Gabriel Bolaos, Jaime Rodriguez y Jos Hoyos

En este proyecto se usar un conjunto de datos de gneros de pelculas. Cada observacin contiene el ttulo de una pelcula, su ao de lanzamiento, la sinopsis o plot de la pelcula (resumen de la trama) y los gneros a los que pertenece.

El algoritmo predice la probabilidad de que una pelcula pertenezca, dada la sinopsis, a cada uno de los gneros.

En primer lugar, probamos el eliminar los stopwords o palabras ms comunes y esto mejor la prediccin del modelo. Seguidamente usamos lematizacin, proceso en el cual cada palabra se busca en un diccionario el cual ha sido previamente calculado y este, para cada palabra, nos va a decir cul debe ser su representacin. Usamos dos tipos de lematizacin, una que da la definicin de la funcin para que tenga como parmetro texto y devuelva una lista de lemas, y el otro para lematizar los verbos en cada comentario. Los mejores valores se dieron con el primero por lo que solo documentamos este.

Para tokenizar los comentarios probamos los tokenizadores Count y TFID con valores por defecto. Los resultados mejoraron usando TFIDVectorizer en lugar de CountVectorizer. Esto es porque mientras que CountVectorizer simplemente cuenta el nmero de veces que aparece una palabra en un texto, TFID no solamente cuenta el nmero de veces sino que tambin evala que tan importante es la palabra en ese texto. Esto se hace gracias a la penalizacin de algunas palabras que el algoritmo considera menos importantes. Precisamente este factor es el que puede darle ventaja de TF-IDF sobre CountVectorizer, dado que, en resumen, este ltimo pesa todas las palabras por igual, mientras que el primero ayuda a lidiar con las palabras ms repetitivas y las penaliza.

En conclusin, usando la mayora de tcnicas de preprocesamiento, los mejores valores predictivos del modelo los encontramoso usando FIDVectorizer + Stop words + Lematizacin de verbos.

Calibración del modelo¶

Las redes neuronales pueden hacer muy facilmente overfitting con los datos de entrenamiento. Por lo que hacer split con data de validacin, permite calibrar hiperparametros de la red y as hacer que el algoritmo logre generalizar todos los casos posibles.

X_train, X_val, Y_train, Y_val = train_test_split(X_train, y_train_genres, test_size=0.15, random_state=42)

Definimos una funcin nn_model_params que crea una red neuronal a partir de 7 diferentes parmetros a calibrar:

In [22]:
# Definicin de funcin que crea una red neuronal a partir de diferentes parmetros (nn_model_params)
def nn_model_params(optimizer,
                    neurons,
                    batch_size,
                    epochs,
                    activation,
                    patience,
                    loss): #Funcin de prdida
    
    K.clear_session()

    # Definicin red neuronal con la funcin Sequential()
    model = Sequential()
    
    # Definicin de las capas de la red con el nmero de neuronas y la funcin de activacin definidos en la funcin nn_model_params
    model.add(Dense(neurons, input_shape=(dims,), activation=activation))
    model.add(Dense(neurons, activation=activation))
    model.add(Dense(output_var, activation=activation))

    # Definicin de funcin de perdida con parmetros definidos en la funcin nn_model_params
    model.compile(optimizer = optimizer, loss=loss)
    
    # Definicin de la funcin EarlyStopping con parmetro definido en la funcin nn_model_params
    early_stopping = EarlyStopping(monitor="val_loss", patience = patience)

    # Entrenamiento de la red neuronal con parmetros definidos en la funcin nn_model_params
    model.fit(X_train, Y_train,
              validation_data = (X_val, Y_val),
              epochs=epochs,
              batch_size=batch_size,
              callbacks=[early_stopping, PlotLossesKeras()],
              verbose=True
              )
     
    return model

En resumen, dado que las redes neuronales pueden hacer facilmente overfitting, partimos la muestra de nuevo para obtener train y test en validacin y poder as generalizar todos los casos posibles.

Luego definimos una funcin para calibrar 7 hiperparmetros de una red neuronal y poder as encontrar los mejores valores.

Para la calibracin de hiperparmetros el modelo Grindsearch, al considerar todas las combinaciones posibles, es un poco mejor que el Ramdonsearch dado que este ltimo toma valores aleatorios y los evala. Por lo tanto, aunque sabemos que es ms costoso computacionalmente, y lo fue, escogimos el Grindsearch para calibrar los hiperparmetros de la red neuronal.

Los mejores valores arrojados son un modelo de activacin sigmoidea, la cul al estar sus valores entre 0 a 1 se usa especialmente para modelos en los que tenemos que predecir la probabilidad, como en este caso.

De igual manera, 20 fueron las pocas ms ptimas, que se explican como cada ciclo de correccin de propagacin hacia atrs y hacia adelante para reducir las prdida. Es curioso que aunque en la grfica se observa que la funcin de prdida no lleg a una estabilidad, los valores predictivos del modelo arrojados son decentes.

El batchsize es 64, este es el nmero de ejemplos que se introducen en la red para que entrene de cada vez. Al haber escogido el menor entre 64 y 128, este entrena ms rpido al tener en memoria menor cantidad de datos.

Se us una funcin de prdida de binarycrossentropy que mide la presentacin de un modelo cuyo rendimiento es la probabilidad en un rango entre 0 y 1.

Por ltimo, 256 neuronas fueron las ms ptimas y un optimizador Adam el cul combina las ventajas de los algoritmos RMSprop y Momentum para mejorar el proceso de aprendizaje de un modelo.
